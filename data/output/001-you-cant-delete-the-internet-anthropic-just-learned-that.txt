It started with a takedown notice. GitHub received it, processed it overnight, and by morning, 8,100 repositories were gone. The code hundreds of thousands of lines of it vanished from the platform. Anthropic had filed a DMCA request over leaked source code for Claude Code, their AI-powered coding assistant.

On paper, this is routine IP enforcement. In practice, it lit a fuse.

By the numbers:

8,100 repos taken down
30,000 stars on the rewrite
512,000 lines permanently in the wild
The Takedown

GitHub's DMCA process under Section 512 of the Digital Millennium Copyright Act is fast. Rights holders file, GitHub responds, repositories go dark. The initial action is automatic and swift.

The code targeted was Claude Code's internals a substantial codebase. The question nobody was asking loudly enough: what happens when the internet has already indexed it?

The Developer Who Woke Up at 4 AM

Sigrid Jin, a Korean developer previously featured by the Wall Street Journal for burning through 25 billion Claude Code tokens in a year, did something remarkable. He woke up at 4 AM, sat down, and rebuilt the core architecture from scratch in Python by sunrise.

He released it as claw-code.

The repository exploded. Thirty thousand GitHub stars. Within the developer community, it became one of the fastest-climbing repositories in recent memory not because of the code alone, but because of what it represented: a direct, technical rebuttal to the idea that you can unpublish something from the internet.

The rough shape of the agent loop looks like this:

import anthropic

def agent_loop(user_input: str, tools: list, history: list = []) -> str:

            max_tokens=8096,

            messages=history

        if response.stop_reason == "tool_use":
            tool_results = dispatch_tools(response.content)
            history.append({"role": "assistant", "content": response.content})
            history.append({"role": "user", "content": tool_results})
        else:
            return response.content[0].text

The Copyright Problem Nobody Wants to Answer

Here's where things get genuinely complicated. Anthropic's own CEO has suggested publicly that large portions of Claude Code were generated by Claude itself.

Can a company hold copyright on code that an AI generated?

In 2025, the DC Circuit Court ruled that AI-generated work does not automatically qualify for copyright protection. The ruling aligns with existing Copyright Office guidance: copyright requires human authorship. If the code was primarily generated by Claude, the legal ground under Anthropic's DMCA claim starts to look shaky.

This isn't just a philosophical argument. If a court were to rule that the code is not copyrightable because it was AI-generated, the entire DMCA action rests on a foundation that may not hold.

The Decentralized Mirror Problem

Even setting the legal question aside, the operational reality is worse for enforcement. The same day the repositories were taken down, mirrors began appearing on decentralized platforms. One, called Gitlawb, went live almost immediately with a stated policy: "Will never be taken down."

This is the core architectural reality of the internet that IP enforcement keeps running into. DMCA is designed for a world where content lives on a few platforms you can send a letter to. That world has not existed for a long time.

512,000 Lines. Permanently Out There.

The leaked codebase sits at roughly 512,000 lines. It has been cloned, mirrored, forked, rebuilt, and compressed into dozens of derivative forms. Some derivatives are functionally indistinguishable from the original. Others, like claw-code, are fresh implementations inspired by the architecture.

Anthropic tried to enforce copyright on code that may not be copyrightable, on a platform that represents a fraction of where the code now lives. The takedown worked on GitHub. It failed everywhere else and it likely guaranteed the code would spread faster than it would have otherwise.

What This Actually Changes

The AI authorship question is now a business risk. If companies are building commercial products on AI-generated code and asserting copyright over them, the DC Circuit ruling creates real exposure. The next enforcement action in this space will likely get litigated rather than settled quietly.

DMCA as a deterrent is weakening. The speed with which claw-code appeared hours, not weeks shows that the developer community's tooling for rapid reconstruction has matured. A takedown is now more likely to produce a rewrite than a retraction.

Decentralized hosting is not theoretical anymore. Gitlawb going live the same day the repos were pulled is a data point, not an edge case. The infrastructure exists and it is being used.

The internet does not have a delete button. It has a DMCA process — which works best when the content is on a single platform, the copyright is unambiguous, and nobody particularly cares about preserving the material.

In this case, none of those conditions were true.

The code is out there. It is not coming back.
